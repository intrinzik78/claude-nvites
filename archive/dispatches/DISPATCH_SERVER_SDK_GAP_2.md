# SDK Gap: Payment Staff Endpoints

**Date:** 2026-03-25
**Workstream:** surface-command-center
**Source:** Slice 6 handoff (server branch, e91d3d0)

## Problem

Four new payment endpoints shipped in Slice 6 but have no hand-written SDK wrappers in `sdk-ts/src/api/`. The TypeScript types were regenerated by `cargo xtask build-all` (they exist in `sdk-ts/src/types/generated.d.ts`), but the endpoint wrappers that surfaces import don't exist yet.

**Endpoints missing wrappers:**
- `GET /v1/bookings/{id}/payments` → `getBookingPayments(bookingId: number)`
- `GET /v1/payments/held` → `getHeldPayments()`
- `POST /v1/payments/{id}/approve` → `approvePayment(paymentId: number)`
- `POST /v1/payments/{id}/decline` → `declinePayment(paymentId: number)`

**Types already generated:**
- `HeldPaymentDto` — composite type with payment fields + booking context
- `PaymentTransactionDto` — full staff record
- `TransactionType`, `TransactionStatus` — enums

## Reasoning

The command center fraud review UI cannot call these endpoints without SDK wrappers. This is the same pattern as the Slice 5 SDK gap (waiver confirmation, call-ahead) — `build-all` regenerates types only, new endpoints need hand-written wrappers + barrel export.

## Proposed Solution

1. Create `sdk-ts/src/api/payments.ts` (or extend existing file if one exists)
2. Add four wrapper functions following the existing pattern in `sdk-ts/src/api/`
3. Add barrel export in `sdk-ts/src/api/index.ts`
4. All four endpoints require auth (Editor) — wrappers must pass the bearer token

**Pattern reference:** See existing wrappers in `sdk-ts/src/api/` for the `client.send::<T>(req)` / fetch-based pattern.

## Confidence

**High.** This is mechanical — the types exist, the endpoints are live, the wrapper pattern is established. No design decisions required.
